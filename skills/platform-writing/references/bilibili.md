# B站 (Bilibili) Content Guide

Knowledge-oriented, detail-rich content for an audience that values depth and respects expertise. B站 viewers will watch 10-15 minute videos if the content is genuinely informative. They also reward personality and humor.

## Voice & Tone
- **知识型但有人味** — 专业 + 幽默，不是干巴巴的教程
- **详细/严谨** — B站观众最烦"水视频"，要干货密度
- **人格化** — 有口头禅、有人设、有梗，观众是冲"你"来的
- **平等交流** — 不是老师讲课，是"我研究了一下，跟你分享"

## Length & Structure
- **视频文案 5-15分钟** — 知识类 8-12分钟是甜区；教程类可到 20分钟
- **脚本字数**: 约 1500-3500字（按 200字/分钟语速算）
- **标题**: 必须有信息量 + 吸引力。B站标题比抖音长，可以更详细：
  - ✅ "我花 100 小时测了 20 个 AI 编程工具，这是我的结论"
  - ✅ "Claude 4.6 深度测评：终于能替代程序员了吗？"
  - ✅ "2026 年了，普通人到底该不该学 AI？我从招聘数据里找了答案"
  - ❌ "AI 编程工具测评" (太干，没有钩子)

## Script structure (B站知识类黄金结构)
```
【开头 30-60s】钩子 + 背景 + 承诺价值
  - "这期视频我准备了 2 周"
  - 点出为什么这个话题值得 10 分钟

【主体 5-12min】分章节（B站可以加进度条章节）
  Part 1: 背景/问题
  Part 2: 核心分析（数据/对比/案例）
  Part 3: 实操/方法论
  Part 4: 进阶/避坑

【结尾 30-60s】总结 + 三连 + 下期预告
  - "如果对你有帮助，求个三连"
  - "下期我们讲 XXX，关注不迷路"
```

## Formatting conventions
- **完整脚本** — 包含画面提示和口播，方便剪辑
- **章节标注** — 【Part 1】【Part 2】对应视频进度条
- **画面/口播分离** — 用"画面:" "口播:" 或类似格式
- **数据可视化提示** — 提到数据时标注"配图表/截图"
- **时间轴** — 关键节点标注时长，方便控时

## Script example (片段)
```
【开头 0:00-0:45】
画面: 电脑前写代码，表情困惑
口播: "我用了 5 年 GitHub Copilot，直到上周试了 Claude Code...说实话，有点颠覆。这期视频，我把 6 款主流 AI 编程工具测了一遍，花了整整两周。如果你在纠结该用哪个，看完这 10 分钟，你应该就有答案了。"

【Part 1: 评测标准 0:45-2:00】
画面: PPT 展示评测维度
口播: "先说我是怎么测的。我用同一组任务——写一个全栈待办应用、修 5 个真实 bug、重构一段祖传代码。评测维度有四个：代码质量、上下文理解、速度、还有最重要的，真能不能帮我省时间..."

【结尾 12:00-12:30】
画面: 总结表格
口播: "总结一下，如果你是新手，我推荐 X；如果你是重度用户，Y 更适合。这期视频做起来真的累，求个三连支持下。下期我会深入讲 Claude Code 的高级用法，关注我不迷路。我们下期见！"
```

## Hook patterns (B站特有)
- "这期视频我准备了 X 天/周" — 暗示内容深度
- "我把所有 X 都测了一遍" — 全面的承诺
- "看完这期，你就不用再找了" — 省时的承诺
- 数据/钱/时间投入先行 — "花了 2000 块/100 小时"

## CTA patterns (B站 = "三连")
- "求三连（点赞+投币+收藏）"
- "弹幕扣 1 表示你看懂了"
- "一键三连，下期更精彩"
- "关注我，每周 X 更新"

## 与抖音脚本的区别
| | 抖音 | B站 |
|---|---|---|
| 时长 | 30-90秒 | 5-15分钟 |
| 深度 | 一个点讲透 | 多个点系统讲 |
| 节奏 | 快、情绪化 | 稳、知识感 |
| 观众心态 | 打发时间 | 主动学习 |
| CTA | 关注/评论 | 三连/弹幕 |

## Common mistakes
- ❌ 太短太水 — B站观众期待深度，5分钟以下的"干货"容易被吐槽
- ❌ 没有人设 — 纯念稿，观众记不住你
- ❌ 像抖音搬运 — 把抖音脚本拉长 ≠ B站内容
- ❌ 数据造假/没验证 — B站观众会扒细节，翻车会很惨
- ❌ 节奏平 — 10分钟一个语调，完播率会很低
- ❌ 没有章节 — 长视频不加进度条章节，观众找不到想看的
